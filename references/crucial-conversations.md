# 关键对话(高压回应)· Simulator 复盘 Reference

> 用途:本次模拟挂载本框架时,Phase 3 专项段按此结构给参考反馈(不打分)。
> 谁读:Agent。给学员的反馈遵循措辞规则(可点名框架名"关键对话",步骤用"对学员表述"栏)。
> 适用场景:高压类 —— 危机处理 / 当众质疑 / 立场拉锯 / 关系冲突(客户情绪上头、对话陷入沉默或对抗)。
> 来源:Patterson, Grenny, McMillan, Switzler, Gregory《Crucial Conversations: Tools for Talking When Stakes Are High》(3rd Edition)。本结构为该方法在**高压销售场景的改编版**(IFB2.0 课程版),非原著线性步骤。内容经改写以符合授权要求。

---

## 一、框架是什么(一句话)

当利益攸关、观点分歧、情绪激烈三者同时出现(典型如客户业务中断后愤怒施压),普通沟通技巧失效;这套"高压回应"按"先接住情绪 → 再讲事实 → 最后建合作"的顺序,帮学员把对抗拉回共同解决问题。

---

## 二、步骤结构(复盘按此顺序)

> 5 步顺序不可乱:情绪没被接住前,讲事实无效。

| 顺序 | 对学员表述(给学员看) | 这一步要做到 | 原著对应(给 Agent,不对学员说) |
|---|---|---|---|
| 0 | 先接住对方 | 让对方直观感到"你收到了他的信息",先恢复安全感再讲内容 | Make It Safe(Mutual Respect)+ 情绪先行 |
| 1 | 只说事实,不贴标签 | 用客观可核对的事实开场,不夹判断、不贴标签 | STATE 的 S(Share Facts) |
| 2 | 说清楚可能的影响 | 讲这些事实对双方意味着什么,把影响摆上台面 | 课程版实战增补(近 SPIN 的"影响";原著 STATE 无此步) |
| 3 | 真诚表态 | 姿态放低、留有余地("我可能不了解全部情况"),降低对方防御 | STATE 的 T(Tell Story)+ Talk Tentatively |
| 4 | 强调共同目的 | 把"你对我"重新框定为"我们一起解决问题" | Mutual Purpose + CRIB |

**步骤与原著的关系(诚实标注)**:
- 第 0 / 1 / 3 / 4 步忠于原著概念。
- **第 2 步"说清楚影响"是课程版的实战增补,原著 STATE 序列里没有。**
- 原著 STATE 后两步(Ask for paths 问对方 / Encourage testing 鼓励反驳)在"我方回应"的单人模拟里弱化,仅作 Agent 判断依据,不单列为步骤。

## 三、怎么从对话里识别"做到 / 没做到"

> Agent 判断每一步时,优先找以下信号;每条反馈都要带模拟里第 N 轮的证据。

### 第 0 步 · 先接住对方
- ✅ 做到:开口先回应对方的情绪/处境("我听到了""这事确实很重要""我理解您现在的着急");在讲任何方案前,先让对方感到被听见。
- ❌ 没做到:对方还在气头上就直接讲技术细节/辩解/报方案;无视对方情绪;用"你先冷静"这类反而激化的话。

### 第 1 步 · 只说事实,不贴标签
- ✅ 做到:用可核对的客观事实("故障 2:47 发生,定位在 DNS 层,修复方案执行中");不带主观判断。
- ❌ 没做到:说"这不是大问题"(判断)、"你们不用担心"(替对方下结论)、"你不懂技术"(贴标签);用故事/解读代替事实。

### 第 2 步 · 说清楚可能的影响
- ✅ 做到:说明事实对双方的实际意义/后续影响,让对方看到你在意他关心的后果。
- ❌ 没做到:只罗列事实,不连接到对方在意的东西;或夸大/掩盖影响。

### 第 3 步 · 真诚表态
- ✅ 做到:用留余地的说法("这是我目前的判断,也许我遗漏了什么""我可能不了解全部情况");表达自信但不把话说死。
- ❌ 没做到:用绝对化保证("绝对不会再出问题""保证没问题"),反而触发对方"找漏洞"心理;或一味示弱没有立场。

### 第 4 步 · 强调共同目的
- ✅ 做到:明确点出双方共享的目标("咱们都想让这个项目尽快回正轨"),把对立框定成"我们 vs 问题"。
- ❌ 没做到:停在"你对我错"的对立框架;只谈各自立场不找共同点;把共同目的说成空话套话。

> **底层判断依据(Agent 参考,不对学员复述术语)**:
> - 共享意义池(Pool of Shared Meaning):双方信息有没有流进同一个池子,还是各说各话。
> - 行为路径(Path to Action):事实→故事→情绪→行动;对方的情绪来自他编的"故事",学员有没有影响到那个故事。
> - 沉默/对抗(Silence/Violence):对方退缩不说 vs 攻击通牒,都是"不安全"的信号。

## 四、专项段措辞示范(给学员看的样子)

> 按步骤顺序;每步下可有多条,各带信号灯;带第 N 轮证据;[可改进]/[踩雷] 带"下次怎么做"。

```
## 专项复盘 · 关键对话(高压回应)

> 这是本次「关键对话」专项训练的表现反馈。下面按这个方法的几个步骤,逐步看你这次做得怎么样。

### 第 0 步 · 先接住对方
- [做得好] 客户第2轮拍桌子说"不用再来了"时,你先回了一句"我知道训练中断这几天给您压力很大",
  没急着辩解 —— 这一步起手对了,先接住了情绪。

### 第 1 步 · 只说事实,不贴标签
- [可改进] 你第4轮说"这其实不算大问题",这是替客户下了判断,他立刻顶回来。
  下次:只讲可核对的事实(故障点、已做的动作、时间线),把"算不算大问题"留给他判断。

### 第 2 步 · 说清楚可能的影响
- [未涉及] 这次你没说明修复进展对他"向 CEO 交代"这件事意味着什么。
  下次:把事实连到他最在意的后果上。

### 第 3 步 · 真诚表态
- [踩雷] 你第6轮说"保证不会再出问题",客户反而更不信(他上次就是被"保证"过)。
  怎么补救:换成留余地的说法,比如"我先不打包票,把我们已经控制住的部分讲清楚"。

### 第 4 步 · 强调共同目的
- [做得好] 收尾你说"咱们的目标是一样的,让训练尽快恢复",把对话从对立拉回了合作。

### 这次最值得带走的
高压下你"先接住情绪"和"找共同目的"做得不错;最该补的是第1步——别在气头上替客户下判断,先铺事实。
```

## 五、术语对照(原名 → 对学员表述)

| 原著/理论说法(不对学员出现) | 对学员说 |
|---|---|
| Make It Safe / 营造安全感 | 先接住对方 / 让他觉得这场谈话是安全的 |
| Mutual Respect / Mutual Purpose | 让他感到你尊重他 / 你们目标是一致的 |
| STATE - Share Facts | 只说事实,不贴标签 |
| Talk Tentatively | 留有余地地说 / 别把话说死 |
| Silence / Violence | 对方不说话躲着 / 对方攻击施压 |
| Pool of Shared Meaning | 把话摊到一起说清楚 |
| Path to Action / Story | 他的情绪是从他的解读来的 |
| CRIB | 一起找双方都认的目标 |

> 框架名"关键对话"可对学员出现(受控例外)。上述机制类术语一律翻成右栏说法。
